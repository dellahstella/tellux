# Dossier des fiches d'hypothèses détaillées

Ce dossier contient les fiches d'hypothèse individuelle quand une hypothèse Tellux développe un protocole de test ou est en cours de validation.

## Quand créer une fiche d'hypothèse ?

Une hypothèse justifie un fichier dédié dans ce dossier si elle :
- A un **protocole de test préliminaire** documenté
- Est **en cours de data collection** (phase 2 ou 3)
- Nécessite une **analyse méthodologique détaillée** (variabilité, biais, design statistique)

Pour les hypothèses en formulation seulement (pas de protocole), rester dans [`../HYPOTHESES.md`](../HYPOTHESES.md) à la racine.

## Convention de nommage

Chaque fiche d'hypothèse doit respecter : `HYPOTHESE_<SLUG>.md`

**Exemples :**
- `HYPOTHESE_SIGNATURE_EM_TERRITORIALE.md`
- `HYPOTHESE_MEGALITHES_GEOMAGNETIC.md`
- `HYPOTHESE_VARIABILITE_INTEGRATION_EM.md`

## Structure de fiche

Utiliser le template [`../_templates/hypothese.md`](../_templates/hypothese.md) comme point de départ.

## Hypothèses actuelles

Hypothèses pré-identifiées (H1-H3) en `../HYPOTHESES.md`. Aucune fiche détaillée créée pour le moment ; fiches seront créées en ce dossier une fois protocoles de test formalisés.
